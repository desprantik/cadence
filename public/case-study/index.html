S Situation

01 — Situation

Online learning is a
$325 billion market
built on good intentions.

220 million people enrol in online courses every year. They subscribe to Udemy, Coursera, Skillshare, YouTube — often all four. Each platform is a silo: its own dashboard, progress tracker, and recommendation engine. No layer exists to unify them.

220M
MOOC learners globally
3–4
avg. active subscriptions per learner
$325B
online education market, 2025

Research across Udemy, Coursera, Skillshare, and YouTube confirmed a split integration landscape: enterprise platforms expose OAuth APIs; consumer platforms vary from Google OAuth (YouTube) to no public API (Skillshare). MCP (Model Context Protocol) emerged as a viable bridge — simulated in the prototype for platforms that support it, with manual entry as fallback.

C Complication

02 — Complication

87% of those courses
are never finished.

The problem isn't content quality. It's that no system connects intent to completion. Learners lose track of what they've enrolled in, lose the thread of what to focus on, and eventually lose motivation entirely.

01
No unified view
Progress lives across 4+ platforms. There is no single place to see what's started, paused, or buried.
02
No priority signal
Learners don't know which course deserves attention today. Lack of structure is the #1 cited reason for abandonment.
03
No momentum
Without streaks, nudges, or micro-wins, self-directed learning defaults to inaction.
04
No intelligence
Platforms recommend more content, never connecting what you've completed to what you should learn next.
Root cause
Self-directed learners have the intent and the subscriptions — but no system to turn scattered enrolments into a coherent, completed learning journey.

Screens

#/ — Dashboard Hero greeting, animated streak/today/total stats, 82% arc gauge, platform chips with mini-rings, stat badges linking to platform drill-downs, course grids with hover actions.
#/session — Byte focus 3 personalized bytes (text, video, quiz), 5-min soft timer ring, step progress, topic pills.
#/library — Library Dual-axis filters (platform × status), progress bars, time-to-complete metadata, course modal on click.

Key flows

01
Connect a platform
Settings → Connect platform → Marketplace → select connector → Connect via MCP (6-step OAuth/MCP handshake) or Add manually (form). Toggle off to disconnect. Remove with confirmation dialog. State persists in localStorage.
02
5-minute byte session
Dashboard CTA → Session page → advance through 3 bytes (quiz allows one retry) → confetti → streak +1, progress bump → celebration overlay on return home with animated stat deltas.
03
Course actions
Hover any course card → bookmark (wishlist) + ⋮ menu: add/remove wishlist, view similar courses, mark complete, not relevant, price drop alert. Wishlist and alerts persist. Click card → course modal with topic-level progress.
04
Discover & filter
Header search → results with library/discover split. Library page filters by platform + status (all, in-progress, completed, saved). Platform drill-down at #/platform/{slug}/{status}. Profile goals shape byte selection and recommendations.
05
MCP tool transparency
Connected platforms show MCP tool tags (2 visible + expand). Hover any tool for a popover describing what it syncs. Refresh triggers simulated sync with shimmer animation. Badges distinguish MCP connected vs Manually added.

♿ A11y

Accessibility

Usable by keyboard,
screen reader, and everyone.

Accessibility was built into the components, not bolted on. Semantic roles, visible focus, sufficient contrast, and motion that respects user preferences.

⌨️

Full keyboard support

Course cards activate with Enter/Space, modals trap focus and close on Escape, and the overlay action menu is fully arrow/tab navigable.

🎯

Visible focus rings

Every interactive element shows a 2px yellow focus outline with offset — never removed, only styled. Selection state is kept separate from focus.

🔊

Semantic & ARIA

Tabs, tablists, checkboxes, menus, and live regions use correct roles. Toasts announce via aria-live="polite"; icon buttons carry descriptive labels.

🌗

Contrast first

Body text on cream and white clears WCAG AA. Muted text stays ≥4.5:1; status never relies on color alone — it pairs with labels and icons.

📏

Readable minimums

No UI text below 12px, generous line-height for body copy, and tap targets sized for touch on the responsive layout.

🎞️

Reduced motion

Confetti, count-ups, and shimmer are quick and non-blocking, with motion kept subtle so the celebration never gets in the way of the task.
